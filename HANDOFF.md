## 1. 한 줄 요약

**교육 주제 키워드를 입력하면, Gemini가 웹 검색으로 트렌드·타사 사례·커리큘럼 등을 조사해
보고서 형태로 정리해 주는 웹 도구**입니다. 파일 하나(`index.html`)로 동작합니다.

- 만든 목적: 교육 기획 담당자가 "근거 있게" 커리큘럼을 설계하도록 돕기 (자세한 배경은 `PRD_교육기획근거도우미.md` 참고)
- 형태: HTML 파일 1개 (서버·빌드·설치 없음)
- AI: Google Gemini API (무료 등급 사용)
- 배포: GitHub Pages (정적 호스팅)

## 3. 사용 방법 (최종 사용자 기준)

1. Google AI Studio(<https://aistudio.google.com>) → **Get API key → Create API key** → `AIza...` 키 복사 (무료)
2. 도구 화면의 **Gemini API 키** 칸에 붙여넣기 (브라우저에 기억됨)
3. **교육 주제 키워드** 입력 (예: `현업 대상 AI 활용`)
4. (선택) 업종/맥락, 조사할 항목 체크, 출력 타입·범위·분량·검색조건 드롭다운 설정
5. **조사하기** 클릭 → 10~40초 후 보고서가 카드 형태로 출력
6. **복사하기** / **인쇄 / PDF 저장** 버튼으로 결과 활용

## 4. 기술 구성 (의도적으로 단순하게)

| 항목 | 내용 | 왜 이렇게 했나 |
|------|------|----------------|
| 파일 | `index.html` 단 하나 | 비개발자가 환경설정 없이 바로 쓰고 배포하기 위함 |
| 의존성 | 없음 (외부 라이브러리 X) | 깨질 부분을 최소화 |
| AI 호출 | 브라우저에서 Gemini API 직접 fetch | 서버 없이 동작 |
| 웹 검색 | Gemini의 `google_search` 그라운딩 도구 | 최신 정보 + 출처 확보 (환각 줄이기) |
| API 키 | 코드에 박지 않고 사용자가 입력 → `localStorage`에만 저장 | 키 유출 방지. 그래서 공개 배포해도 안전 |

> **중요한 보안 원칙**: API 키는 절대 코드에 하드코딩하지 않습니다. 사용자가 화면에서 입력하고,
> 그 브라우저의 `localStorage`(키 이름 `gemini_api_key`)에만 저장됩니다. 우리 쪽 서버로 전송되지 않습니다.

## 6. 코드 구조 (`index.html` 내부)

파일은 `<style>`(디자인) + `<body>`(화면) + `<script>`(동작) 세 부분입니다.

### 6-1. 화면(HTML)의 칸 구성
- API 키 입력 카드
- 주제 키워드 / 업종 입력 카드
- 옵션 카드: **조사할 항목 체크박스** + 드롭다운 5종(출력타입/범위/분량 · 기간/규모·유형 · 직급/규모)
- 결과 카드: `#report`(보고서) + `#sources`(검색 출처) + 복사/인쇄 버튼

### 6-2. 자바스크립트 핵심 요소 (여기를 고치면 동작이 바뀜)

| 위치 | 역할 |
|------|------|
| `const MODELS = [...]` | 사용할 Gemini 모델 목록. 혼잡하면 순서대로 자동 전환. **모델명 바뀌면 여기만 수정** |
| `callGemini(...)` | 실제 API 호출 함수 (엔드포인트·요청 형식) |
| `extractGemini(data)` | 응답에서 본문 텍스트 + 검색 출처 링크 추출 |
| `const ITEMS = [...]` | **조사 항목 목록** (체크박스가 여기서 자동 생성됨) |
| `SCOPE_TEXT` / `DEPTH_TEXT` / `OUTTYPE_TEXT` | 드롭다운 선택값 → 프롬프트 문구 매핑 |
| `PERIOD_TEXT` / `SIZE_TEXT` / `LEVEL_TEXT` / `SCALE_TEXT` | 검색조건 드롭다운 → 프롬프트 문구 |
| `buildPrompt(...)` | **선택값을 모아 Gemini에 보낼 프롬프트(질문지)를 조립** |
| `renderMarkdown(md)` | Gemini가 준 마크다운을 보고서 HTML(카드·표·불릿)로 변환 |
| `renderChart(raw)` | "시간배분" 코드블록 → 막대그래프 |
| `newsItem(t)` | 뉴스 제목을 "구글 검색 링크"로 변환 |
| `research()` | 버튼 클릭 시 전체 흐름(호출→재시도→렌더링) 담당 |

### 6-3. 동작 흐름 (한눈에)
```
[조사하기 클릭]
  → buildPrompt(): 체크된 항목 + 드롭다운 조건으로 질문지 생성
  → callGemini(): Gemini API 호출 (혼잡 시 자동 재시도 → 보조 모델 전환)
  → extractGemini(): 본문 + 출처 추출
  → renderMarkdown(): 보고서 카드로 그리기 / renderChart(): 그래프
  → #sources: 검증된 출처 링크 표시
```

## 7. 자주 하는 수정 방법 (레시피)

### (A) 조사 항목 추가/삭제
`ITEMS` 배열에 한 줄 추가하면 체크박스가 자동 생성됩니다.
```js
{ id: "고유아이디", label: "화면에 보일 이름", text: "프롬프트에 들어갈 설명", def: true 또는 false },
```
- `def: true`면 기본 체크됨.
- 특별한 형식(표/번호목록)이 필요하면 `buildPrompt` 안의 `~Rule` 패턴을 참고해 추가.

### (B) 드롭다운(검색 조건) 추가
1. HTML에 `<select id="새이름">...</select>` 추가
2. JS 상단에 `const 새El = $("새이름");`
3. 매핑 객체(`예: NEW_TEXT = {...}`) 추가
4. `buildPrompt`의 `filters` 부분에 한 줄 추가

### (C) Gemini 모델 변경
`const MODELS = ["gemini-2.5-flash", "gemini-2.0-flash"];` 에서 모델명만 교체.

### (D) 프롬프트(말투·형식) 수정
`buildPrompt`의 return 템플릿 문자열에서 `[출력 형식]`·`[정확성 규칙]` 부분을 수정.

### (E) 출력 형식 특수 처리
- 표로 받고 싶으면: `curriculumRule`/`timelineRule`처럼 마크다운 표를 지시 → `renderMarkdown`이 표로 그림
- 그래프: `chartRule`(시간배분 코드블록) → `renderChart`가 막대로 그림

## 8. 배포 / 업데이트 방법 (GitHub Pages)

이미 배포돼 있습니다. **코드를 고친 뒤 사이트에 반영하려면:**

1. GitHub 저장소 → `index.html` 열기 → 연필(✏️) 아이콘 또는 **Add file → Upload files**로 새 `index.html` 덮어쓰기
2. **Commit changes**
3. 1~2분 뒤 공개 URL 새로고침하면 반영됨

처음부터 다시 배포하는 경우:
- 저장소 생성(Public) → `index.html` 업로드 → **Settings → Pages → Branch: main /(root) → Save** → 생성된 `https://<아이디>.github.io/<저장소>/` 주소 공유

## 9. 알려진 제약 / 주의사항

| 항목 | 내용 |
|------|------|
| **API 키는 각자 발급** | Gemini 무료 등급. Gemini 키는 `AIza`로 시작. (OpenAI `sk-...` 키와 혼동 주의) |
| **무료 한도/혼잡** | 무료 등급은 분당 요청수 제한·일시적 혼잡이 있음 → 자동 재시도/보조모델 전환으로 완화했지만, 계속 막히면 잠시 후 재시도 |
| **환각(없는 정보)** | AI가 사실을 지어낼 수 있음. 그래서 본문의 모델 링크는 클릭 비활성화했고, **실제 링크는 맨 아래 "검색 출처"에서만** 제공. 회사 사례·수치는 사용자가 한 번 더 확인 권장 |
| **뉴스 링크** | 뉴스 제목 클릭 시 그 제목으로 "구글 검색"이 열림(404 방지). 검색 첫 결과가 해당 기사 |
| **모델명 변동 가능** | Gemini 모델명이 바뀌면 `MODELS` 수정 필요. 오류 메시지가 화면에 그대로 표시되도록 해둠 |
| **CORS** | 현재 Gemini는 브라우저 직접 호출이 됨. 만약 막히면(다른 제공자로 바꾸는 등) 작은 서버가 필요할 수 있음 |
| **공유 주의** | 키가 코드에 없으니 공개 배포는 안전. 단, 회사 대외비 문구를 코드/저장소에 넣지 말 것 |

## 10. 그동안의 작업 이력 (어떻게 여기까지 왔나)

1. **PRD 작성** — "교육 기획 근거 도우미" 기획 (`PRD_교육기획근거도우미.md`)
2. **MVP v1** — 현업 설문 응답을 정리하는 단일 화면 (Claude API) 로 시작
3. **방향 전환** — "키워드로 트렌드·타사 사례·커리큘럼 조사"로 기능 변경
4. **유료 회피** — 별도 API 과금 문제로, 처음엔 "프롬프트 생성기"(claude.ai에 붙여넣기)로 전환
5. **Gemini 직접 연결** — 무료 + 웹 검색이 되는 Gemini API로 직접 호출하도록 구현
6. **선택지/필터 추가** — 조사 항목 체크박스, 출력 타입·범위·분량 드롭다운
7. **가독성 개선** — 줄글 → 짧은 불릿 / 보고서 카드(대시보드) 레이아웃 / 인쇄·PDF
8. **항목 구체화** — 커리큘럼(표), 시간 배분(막대그래프), 실행 순서(번호목록), 뉴스(올해만+검색링크)
9. **안정화** — 혼잡 시 자동 재시도 + 보조 모델 전환
10. **링크 환각 차단** — 본문 모델 링크 비활성화, 검증된 출처 링크만 클릭 가능
11. **검색 조건 4종 + 단계별 일정(타임라인) 추가**
12. **GitHub Pages 배포 완료**

> 참고: 중간에 OpenAI(ChatGPT) API로 잠깐 바꿨다가 무료 문제로 Gemini로 되돌렸습니다.
> 현재 코드에는 OpenAI 흔적이 없습니다(전부 Gemini).

## 11. 향후 개선 아이디어 (선택)

- **공용 키 + 작은 서버**: 팀원이 각자 키 입력하는 대신, 키를 숨기는 백엔드(예: Cloudflare Workers/Netlify Functions)를 두면 키 1개로 운영 가능 (단, 비용이 한 곳으로 몰림)
- **결과 저장/불러오기**: 현재는 새로고침하면 사라짐. 브라우저 저장 또는 파일 내보내기 추가 가능
- **항목 프리셋**: "경영진 보고 세트", "실무 기획 세트"처럼 체크 조합을 버튼 하나로
- **다른 제공자 지원**: 제공자별 호출 함수를 분리하면 모델 교체가 쉬움

## 12. 막혔을 때 디버깅 요령

- 결과 칸에 **"오류가 났어요: ..."** 메시지가 그대로 표시됨 → 그 문구로 원인 파악
  - `API key not valid` / `API_KEY_INVALID` → 키 문제 (AIza 키 맞는지)
  - `quota` / `rate limit` → 무료 한도/혼잡 → 잠시 후 재시도
  - `model ... not found` → 모델명 변경됨 → `MODELS` 수정
  - `CORS` → 브라우저 직접 호출 차단 → 서버 필요
- 브라우저 **F12 → Console** 탭에서 더 자세한 오류 확인 가능
